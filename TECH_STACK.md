# Project Structure

The project follows a modular architecture.

Each module should have a single responsibility.

No circular dependencies should exist.

---

# Dependency Rules

Always prefer

- Stable releases
- Well-maintained libraries
- Lightweight dependencies

Avoid

- Deprecated libraries
- Unmaintained packages
- Experimental packages unless necessary

---

# Security Standards

Never hardcode

- API Keys
- Database Passwords
- Tokens
- Secrets

All secrets must come from:

.env

---

# Configuration Strategy

All configurable values must remain outside the source code.

Examples

- API Keys
- Database URL
- Redis URL
- Model Name
- Temperature
- Chunk Size
- Embedding Model

---
